制約別使用可能アルゴリズム(あくまで目安)

N <= 10～20
　・O(N!)→順列全探索(N<= 10まで)
　・O(2^N)→bit全探索(間に合わなければ再帰関数で枝狩りなど、半分全列挙もあり)
　・O(N^4)など→多重for文による全探索

N <= 100
　・O(N^3)→3重for文全探索

N <= 1000
　・O(N^2)→二重for文全探索
　・O(N^2 logN)→for文内でsort、二分探索(クエリ問題など)

N <= 2*10^5(最も多いパターン)
　・O(NlogN)→sortからの二分探索
　・O(N)→メモを作っておくなどして計算量を削減することも多い

N <= 10^7
　・O(N)→線形探索(累積和、尺取りなど)

それ以上(N >= 10^7)
　・O(logN)→二分探索(sort済み)
　・O(1)→数学
